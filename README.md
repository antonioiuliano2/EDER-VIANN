    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-eder-viann" class="anchor" aria-hidden="true" href="#eder-viann"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>EDER-VIANN</h1>
<p>Emulsion Data Event Reconstruction - Vertex Identification by using Artificial Neural Networks.
Release 1.</p>
<p>This README just serves as a very short user guide, the documentation will be written much later.</p>
<h2><a id="user-content-installation-steps" class="anchor" aria-hidden="true" href="#installation-steps"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Installation steps</h2>
<ol>
<li>
<p>pip3 install --upgrade pip</p>
<p>This step is highly recommended in order to avoid possible problems with intallation of other packages.</p>
</li>
<li>
<p>pip3 install tensorflow==1.14.0 --user</p>
</li>
<li>
<p>pip3 install keras==2.3.1 --user</p>
</li>
<li>
<p>pip3 install tensorflow-gpu==1.14.0 --user</p>
<p>Only required if you have intent to create/train CNN models</p>
</li>
<li>
<p>pip3 install pandas</p>
<p>Pandas are used extensively in this package for routine data manipulation.</p>
</li>
<li>
<p>pip3 install psutil</p>
<p>This utility helps to monitor the script memory usage</p>
</li>
<li>
<p>pip3 install 'h5py==2.10.0' --force-reinstall</p>
<p>Downgrading h5py in order to make it work with the existing version of the Tensorflow.</p>
</li>
<li>
<p>go to your home directory on AFS where you would like to install the package</p>
</li>
<li>
<p>git clone <a href="https://github.com/FilipsFedotovs/EDER-VIANN/">https://github.com/FilipsFedotovs/EDER-VIANN/</a></p>
</li>
<li>
<p>cd EDER-VIANN/</p>
</li>
<li>
<p>python3 setup.py</p>
</li>
<li>
<p>The installation will require another directory, please enter the location on EOS where you would like to keep data and the models
Has to provide up to 10-100 GB of storage depending on whether particular components of the framework is used. An example of the input is /eos/user// . In theory AFS work location also can be specified but it is not recommended.</p>
</li>
<li>
<p>The installer will copy and analyse existing data and the pre-trained model, it might take 5-10 minutes.</p>
</li>
<li>
<p>if the message 'EDER-VIANN setup is successfully completed' is displayed, it means that the package is ready for work</p>
</li>
</ol>
<h2><a id="user-content-additional-info" class="anchor" aria-hidden="true" href="#additional-info"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Additional info</h2>
<ol>
<li>It is recomended to run those processes on lxplus in the tmux shell as some scripts can take up to several hours to execute.</li>
<li>The script name prefixes indicate what kind of opeartions this script perform: R is for acrual reconstruction routines, E for evaluation and M for model cration and training.</li>
<li>In general the numbers in prefixes reflect the order at which scripts have to be executed e.g: R1, R2,R3...</li>
<li>--help argument provides all the available run arguments of a script and its purpose.</li>
<li>The output of each script has the same prefix as the script that generates it. If script generates a temporary output for another script it will have the double prefix e.g: R2_R3 etc.</li>
<li>The files that are the final output have names with capital letters only such as: R5_REC_VERTICES
Those files are not deleted after execution. If not all letters in the file are capitalised that means that the file is temporary and will be eventually deleted by the package once it is not needed anymore.</li>
<li>The screen output of the scripts is colour coded:
<ul>
<li>White for routine operations</li>
<li>Blue for the file and folder locations</li>
<li>Green for successful operation completions</li>
<li>Yellow for warnings and non-critical errors.</li>
<li>Red for critical errors.</li>
</ul>
</li>
<li>Once the program successfully executes it will leave a following message before exiting:
"###### End of the program #####"</li>
</ol>
<h2><a id="user-content-vertex-reconstruction" class="anchor" aria-hidden="true" href="#vertex-reconstruction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Vertex Reconstruction</h2>
<ol>
<li>
<p>Please make sure that you have a file with hits that there were reconstructed as Tracks.
Following columns are required:</p>
<ul>
<li>Track ID Quadrant or Event ID (if MC truth track reconstruction data is used)</li>
<li>Track ID (FEDRA or MC depending which data is used)</li>
<li>x-coordinates of the track hits</li>
<li>y-coordinates of the track hits</li>
<li>z-coordinates of the track hits</li>
</ul>
</li>
<li>
<p>Please open $AFS/EDER_VIANN/Code/Utilities/Parameters.py and check that the lines between 6-13
Within the list of naming conventions correspond to headers in the file that you intend to use.</p>
</li>
<li>
<p>Check the 'CNN_Model_Name' variable - it has the name of the Model that is used for reconstruction (included in the package). If you wish to use your own,        please place it in the $EOS/EDER_VIANN/Models and change the 'CNN_Model_Name' variable accordingly. You might need to change resolution and MaxX, MaxY, MaxZ      parameters if the model was trained with images that have had different size because the model might fail.</p>
</li>
<li>
<p>If happy, save and close the file.</p>
